# SkillPool 开源路线图

## 1. 目标

把当前本地实战工具整理成一个适合发布到 GitHub 的开源项目。

目标不是一次性“完美发布”，而是分阶段降低私有路径、个人状态和环境耦合。

## 2. 当前已经具备的基础

- 有清晰的核心目录结构
- 有 CLI
- 有本地 Web 控制台
- 有测试
- 有中文文档
- 有盘点、报告、发布、回滚等完整工作流

## 3. 当前还不适合直接公开的点

- 目录默认值仍然绑定个人路径
- 运行态数据目录在仓库根下
- 文档仍以个人环境为主
- 许可证尚未选定
- 还缺少更正式的项目对外说明与贡献边界

## 4. 建议分阶段推进

### 阶段 A：仓库整理

- 增加 `.gitignore`
- 增加 `CONTRIBUTING.md`
- 增加 `SECURITY.md`
- 补正式操作手册与维护手册
- 把 README 变成入口索引

### 阶段 B：配置去个人化

- 引入更清晰的示例默认配置
- 把个人绝对路径迁移为“示例配置 + 本地覆盖”
- 减少仓库源代码对个人环境的硬编码印象

### 阶段 C：发布友好化

- 增加英文 README 或双语摘要
- 增加开源许可证
- 增加 issue / PR 模板
- 增加版本号与 changelog 策略

### 阶段 D：功能扩展

- MCP 配置管理
- 技能池清理工作流
- 更强的导出和筛选

## 5. 许可证

当前未自动替你选择许可证。

原因：

- 许可证会直接决定别人能否商用、修改、闭源分发
- 这属于有后果的对外决策

建议你后续在这些方向里选一个：

- `MIT`
- `Apache-2.0`
- `GPL-3.0`

在未选定前，先把仓库结构整理好是更稳的做法。

## 6. 下一步最推荐动作

如果继续朝 GitHub 开源推进，最值得马上做的是：

1. 把默认客户端路径抽成示例配置
2. 补 issue / PR 模板
3. 选许可证
4. 增加英文首页摘要
