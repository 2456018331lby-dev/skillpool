# Code of Conduct

## Our Goal

SkillPool should be a calm, practical, respectful project for maintainers and contributors.

## Expected Behavior

- Be respectful and specific.
- Prefer evidence over assumption.
- Explain risks and tradeoffs clearly.
- Focus on improving the tool, docs, and contributor experience.

## Unacceptable Behavior

- Harassment or personal attacks
- Bad-faith disruption
- Publishing sensitive local information without consent

## Scope

This code of conduct applies to project discussions, issues, pull requests, and review conversations.

## Maintainer Guidance

Maintain a practical and proportionate response. Protect contributors first, then restore productive collaboration.
